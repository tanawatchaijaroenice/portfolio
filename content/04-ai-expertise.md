# AI-Augmented Engineering — ทักษะการทำงานร่วมกับ AI

> ใช้สำหรับ section: **expertise** (highlight block) หรือ section ใหม่แยกต่างหาก
> นี่คือ **จุดขายเด่น** — แสดงความเป็น "AI-augmented engineer" ที่บริษัทยุคนี้ตามหา

## หัวข้อหลัก / Headline
**TH:** ใช้ AI เป็นเครื่องมือทำงานอย่างเป็นระบบ ไม่ใช่แค่ถาม-ตอบ
**EN:** I use AI as a systematic engineering tool — not just Q&A.

## เครื่องมือ / Tools
- **Claude** (Anthropic) — coding assistance & agentic workflows
- **Gemini** (Google) — multimodal analysis & logic verification
- **Codex** (OpenAI) — code completion & code translation

## วิธีใช้งานจริง / How I use AI
1. **เร่งงาน / Scaffold** — ให้ AI ร่าง component/function แล้วปรับต่อ
2. **Debug / หา root cause** — ใช้ AI ช่วยไล่ bug และวิเคราะห์ error
3. **ออกแบบ / ปรึกษา architecture** — ถามแนวทาง, review โค้ด, best practice
4. **Prompt engineering & agent ที่เป็นระบบ** — วาง prompt และใช้ AI agent ทำงานหลายขั้นตอนต่อเนื่อง
5. **สร้าง reusable skill files** — สร้างไฟล์ skill ที่นำกลับมาใช้ซ้ำได้
6. **สร้าง knowledge base ของฟีเจอร์** — เขียน knowledge base ของแต่ละฟีเจอร์เพื่อให้ทำงานต่อ/ต่อยอดได้
7. **วาง action plan ให้ AI ลงมือได้แม่นยำ** — แตกงานเป็นแผนให้ AI execute ได้อย่างแม่นยำ

## EN summary (สำหรับ resume)
> AI-augmented frontend engineer. I integrate AI tools (Claude, Gemini, Codex) systematically into my workflow: scaffolding, root-cause debugging, architecture consulting, and prompt engineering. I build reusable AI "skills," per-feature knowledge bases, and structured action plans so AI agents can execute reliably — turning AI from a chat assistant into a dependable part of my engineering process.

## ประสบการณ์ AI ในโปรดักต์ / AI in products
- เคยลองเรียก **LLM API** ในโปรเจคตัวเอง/ทดลอง
- พัฒนา side project (LifeOS, Next.js) ร่วมกับ AI ตั้งแต่ต้นจนจบ

## จุดที่ทำให้ต่าง / Why this matters
**TH:** ผมไม่ได้ใช้ AI แค่ generate โค้ดมั่วๆ — ผมวางระบบให้ AI ทำงานแม่นยำ ตรวจสอบผลลัพธ์ด้วยหลักการ root-cause เดียวกับที่ใช้ debug ทำให้ได้ทั้งความเร็วและคุณภาพ
**EN:** I don't just let AI generate code blindly — I build the scaffolding (skills, knowledge bases, action plans) that makes AI output accurate, and I verify results with the same root-cause discipline I use for debugging. That gives me both speed and quality.
